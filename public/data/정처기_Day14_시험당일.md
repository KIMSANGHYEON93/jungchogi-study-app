## 시험장 도착 전 — 최종 30분 루틴

### 순서대로 따라하기

```
1. [5분] CARD 4 암기법 모음 — 소리 내어 읽기
   도분명확 / 식통감기 / 객동기 / 단통시인
   자스제외공내 / 우논시절통순기 / 상점비환 / 도부이결다조

2. [5분] CARD 1 코드 5대 규칙 — 머릿속 시뮬레이션
   *p+2 ≠ *(p+2) / a++ vs ++a / 변수=부모 메서드=자식
   == vs equals / 슬라이싱 e 미포함

3. [5분] CARD 2 SQL 실행순서 — 손으로 쓰기
   FROM→WHERE→GROUP BY→HAVING→SELECT→ORDER BY
   DELETE vs TRUNCATE vs DROP 표

4. [5분] CARD 5~6 디자인패턴 + 보안 — 훑어 읽기
   빌프팩앱싱 / 어데퍼브플컴프 / Observer/Strategy
   AES=대칭 RSA=비대칭 / XSS DDoS IDS IPS

5. [5분] CARD 7~8 UML + 정렬 — 마무리
   include=필수 extend=선택 / 전위VLR 중위LVR 후위LRV
   안정정렬=버삽병

6. [5분] 심호흡 + 물 마시기 + 화장실
   → 시험 시작!
```

## 시험 중 전략 (150분 운영법)

### 시간 배분

```
┌─────────────────────────────────────────────┐
│  구간 1 [0~50분]  전체 훑기 + 쉬운 문제     │
│  ├ 1~20번 전체 한 번 읽기                    │
│  ├ 확실한 문제 먼저 답 작성                   │
│  └ 모르는 문제 ★ 표시 후 넘기기              │
│                                              │
│  구간 2 [50~120분]  ★ 문제 집중 풀기         │
│  ├ 코드 트레이싱 = 종이에 변수 표 그려서 추적│
│  ├ SQL = 단계별 중간 결과 테이블 작성         │
│  └ 이론 = 암기법 연상 → 빈칸 채우기          │
│                                              │
│  구간 3 [120~150분]  검토 + 마무리           │
│  ├ 모든 답안 빠짐없이 작성했는지 확인         │
│  ├ 코드 문제 결과값 재검증                    │
│  ├ 오타/탈자 점검 (대소문자 주의!)           │
│  └ 빈칸 절대 남기지 않기 (부분점수!)         │
└─────────────────────────────────────────────┘
```

### 핵심 실전 규칙 7가지

```
① 빈칸은 절대 남기지 마라
   → 모르더라도 연관 키워드 적기 (부분점수 가능)

② 코드 문제는 반드시 변수 추적표를 그려라
   → 머릿속으로만 하면 실수한다
   → 종이 여백에 변수명 | 값 표를 그리기

③ SQL 문제는 중간 테이블을 그려라
   → FROM→WHERE 결과 → GROUP BY 결과 → HAVING 결과 순서대로

④ 대소문자에 주의하라
   → Java: System.out.println (S 대문자, o 소문자)
   → Python: True/False (첫글자 대문자)
   → SQL: NULL (대문자)

⑤ 출력 형식을 정확히 파악하라
   → print(a, b) vs print(a, end="") → 공백/줄바꿈 차이
   → sep, end 파라미터 확인

⑥ "~을 쓰시오" 유형은 정확한 용어로 답하라
   → "싱글턴" (X) → "Singleton" 또는 "싱글톤 패턴" (O)
   → 영문 철자 주의

⑦ 약술형은 키워드 중심으로 간결하게
   → 핵심 키워드 2~3개 포함하면 만점
```

## 답안 작성 팁

### 코드 트레이싱 답안 예시

```
문제: 출력 결과를 쓰시오

[내 풀이 과정 - 여백에]
a=5, b=3
a++ → a 사용(5) 후 증가 → a=6
++b → b 증가(4) 후 사용

[답안지에 쓸 것]
→ 결과값만 깔끔하게: 6 4
```

### 용어 답안 예시

```
문제: 다음 설명에 해당하는 용어를 쓰시오
"인스턴스를 하나만 생성..."

[답안지에 쓸 것]
→ 싱글톤(Singleton) 패턴
   (한글+영문 병기하면 안전)
```

### SQL 답안 예시

```
문제: 실행 결과를 쓰시오

[풀이 과정 - 여백에]
① FROM employee → 전체 5행
② WHERE salary > 3000 → 3행
③ GROUP BY dept → 2그룹
④ HAVING COUNT(*) >= 2 → 1그룹

[답안지에 쓸 것]
→ 결과 테이블 형태로:
  dept | count
  Dev  | 2
```
