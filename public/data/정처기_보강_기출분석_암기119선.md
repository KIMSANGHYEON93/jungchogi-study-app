### 보강 1: C언어 서식문자열 & 제어문자 [암기 001~002]

```
서식문자열:
%d  정수 10진수     %f  실수(소수점)
%o  정수 8진수      %e  지수형 실수
%x  정수 16진수     %c  문자 1개
%s  문자열          %p  주소(16진수)
%u  부호없는 정수   %ld long형 10진수

제어문자:
\n  줄바꿈(new line)     \t  탭(일정 간격)
\b  백스페이스           \r  커서를 줄 처음으로(carriage return)
\0  널 문자              \\  역슬래시
\'  작은따옴표           \"  큰따옴표
\a  벨 소리(alert)       \f  페이지 넘김
```

**자주 나오는 함정**:
- `\r` = 커서를 줄 처음으로 이동 (줄바꿈 아님!)
- `printf("ABC\rDE")` → 출력: `DEC` (ABC 위에 DE 덮어쓰기)

### 보강 2: 연산자 우선순위 [암기 003]

```
높음 ← 단항(!,~,++,--,sizeof)
       산술(*, /, %)  →  산술(+, -)
       시프트(<<, >>)
       관계(<, <=, >=, >)  →  관계(==, !=)
       비트(&)  →  비트(^)  →  비트(|)
       논리(&&)  →  논리(||)
       삼항(?:)
       대입(=, +=, -=, *=, /=)
낮음 ← 콤마(,)

암기: 단산시 관비논 삼대콤
      (단항→산술→시프트→관계→비트→논리→삼항→대입→콤마)
```

### 보강 7: 데이터 모델 & 이상현상 [암기 025~028]

```
데이터 모델 구성: 개체(Entity) + 속성(Attribute) + 관계(Relationship)
데이터 모델 종류: 개념적 → 논리적 → 물리적
데이터 모델 표시 요소: 구조(Structure) + 연산(Operation) + 제약조건(Constraint)

이상(Anomaly) = 데이터 중복으로 인한 조작 문제:
  삽입 이상: 원하지 않는 값까지 삽입해야 함
  삭제 이상: 의도와 상관없는 값도 함께 삭제 (연쇄삭제)
  갱신 이상: 일부만 갱신되어 불일치 발생
  → 해결: 정규화!
```

### 보강 8: 함수적 종속 & 반정규화 [암기 029, 031]

```
함수적 종속 X→Y: X 값이 정해지면 Y 값이 유일하게 결정
  X = 결정자(Determinant), Y = 종속자(Dependent)

완전 함수적 종속: X 전체에 종속 (부분집합에는 종속 X)
부분 함수적 종속: X의 부분집합에도 종속 → 2NF에서 제거

이행적 함수적 종속: A→B, B→C이면 A→C → 3NF에서 제거

반정규화(Denormalization): 성능 향상을 위해 의도적 정규화 위반
  테이블 통합: 조인 빈번한 경우
  테이블 분할: 수평(레코드 기준) / 수직(속성 기준)
  중복 테이블/속성 추가: 조회 경로 단축
```

### 보강 9: 인덱스 & 뷰 & 트랜잭션 & 파티션 [암기 032~036]

```
인덱스(Index):
  <키값, 포인터> 쌍의 데이터 구조
  클러스터드: 데이터 자체가 키 순서대로 정렬 저장
  넌클러스터드: 인덱스 키만 정렬, 실제 데이터 미정렬
  ★ CREATE INDEX idx_name ON student(name);

뷰(View):
  하나 이상의 기본 테이블에서 유도된 가상 테이블
  물리적으로 존재하지 않음, 논리적 독립성 제공
  CREATE VIEW로 생성, DROP VIEW로 제거

트랜잭션 ACID:
  Atomicity(원자성): 전부 반영 or 전부 취소
  Consistency(일관성): 완료 후 일관된 상태
  Isolation(독립성): 동시 실행 시 간섭 불가
  Durability(지속성): 완료 결과 영구 반영

파티션:
  범위 분할(Range): 지정 열 값 기준
  해시 분할(Hash): 해시 함수 결과에 따라
  조합 분할(Composite): 범위 + 해시
```

```
    결정 커버리지: 모든 분기 T/F
    조건 커버리지: 각 조건 T/F
    조건/결정 커버리지: 조건+결정 모두
    변경 조건/결정(MC/DC): 각 조건이 독립적으로 결과 영향
    다중 조건 커버리지: 모든 조건 조합

  종류:
    기초 경로 검사: 논리적 복잡성 측정
    조건 검사: 논리적 조건 테스트
    루프 검사: 반복 구조 초점
    데이터 흐름 검사: 변수 정의/사용 위치 초점

★ 블랙박스 테스트 (기능 테스트, 외부 명세 기반):
  동치 분할(동등 분할): 유효/무효 그룹핑 → 대표값
  경계값 분석: 경계값 부분 집중
  결정 테이블: 조건+행위 조합 테이블
  상태전이: 상태 간 전이 경우의 수
  유스케이스: 실제 사용 프로세스 흐름
  분류트리: 트리 구조 분석
  페어와이즈: 입력 매개변수 쌍별 조합
  원인-효과 그래프: 입출력 관계 분석
  오류 예측: 경험/감각 기반
  비교 검사: 여러 버전 동일 입력 비교
```

### 보강 16: 비용 산정 기법 [암기 083~084]

### 보강 17: DoS 공격 유형 상세 [암기 085]

```
★ Ping of Death: 허용 범위(65,536B) 초과 패킷 전송 → 네트워크 마비
★ Smurfing(스머핑): IP/ICMP 악용, 대량 데이터 집중 전송
★ SYN Flooding: 3-way-handshake 의도적 중단 → 서버 대기 상태
★ TearDrop: Fragment Offset 변경 → 패킷 재조립 오류 → 과부하
★ Land: 송신 IP = 수신 IP (자기 자신에게 무한 응답)
★ DDoS: 분산된 공격 지점에서 한 곳 공격 (좀비PC → 에이전트)
```

```

무작위 대입(Brute Force): 모든 값 대입

좀비 PC: 악성코드 감염된 원격 조종 PC

봇넷(Botnet): 감염 PC들의 네트워크

★ 제로데이 공격: 취약점 공표 전 공격

★ 랜섬웨어: 파일 암호화 → 금전 요구
★ 백도어(Back Door): 비밀 통로 (탐지: 무결성검사/로그분석/SetUID)
```

### 보강 19: 암호화 알고리즘 상세 [암기 089~090]

```
대칭키 (개인키, 같은 키):

  AES:  2001 미국 NIST, 128비트 블록, 키 128/192/256

비대칭키 (공개키+비밀키):

  SHA 시리즈: 미국 NSA 제작, NIST 표준 (가장 많이 사용)

  SNEFRU: R.C.Merkle, 128/254비트
```

### 보강 20: OS & 스케줄링 상세 [암기 091~095]

```
운영체제 목적: 처리능력↑ 반환시간↓ 사용가능도↑ 신뢰도↑

UNIX 구성:
  커널(Kernel): 핵심, 프로세스/기억장치/파일/입출력 관리
  쉘(Shell): 명령어 해석기, 사용자↔시스템 인터페이스
    종류: Bourne Shell, C Shell, Korn Shell

★ 페이지 교체 알고리즘:
  OPT: 앞으로 가장 오래 안 쓸 페이지 교체 (최적, 실현 어려움)
  FIFO: 가장 먼저 들어온 페이지 교체
  LRU: 최근 가장 오래 안 쓴 페이지 교체
  LFU: 사용 빈도 가장 적은 페이지 교체
  NUR: 최근 미사용 페이지 (참조비트+변형비트)
  SCR: FIFO 보완, 자주 쓰는 페이지 보호

★ 비선점 스케줄링:
  FCFS(FIFO): 도착 순서대로
  SJF: 실행시간 짧은 것 먼저
  HRN: SJF 보완, 우선순위 = (대기시간+서비스시간)/서비스시간
  기한부(Deadline): 일정 시간 내 완료
  우선순위(Priority): 우선순위 높은 것 먼저

★ 선점 스케줄링:
  SRT: SJF의 선점 형태 (잔여시간 짧은 것)
  라운드 로빈(RR): 시분할, 시간 할당량(Quantum)만큼 실행 후 교대
  다단계 큐(MQ): 그룹별 다른 준비 큐
  다단계 피드백 큐(MFQ): 큐 간 이동 가능 (MQ 개선)

에이징(Aging): 기다린 시간 비례하여 우선순위↑ (기아 방지)
```

```
  NULL 무결성: 특정 속성 NULL 불가
  고유 무결성: 특정 속성 값 서로 다름
  키 무결성: 최소 1개 키 존재
```

### 보강 22: 네트워크 상세 [암기 104~113]

```
★ IPv6:
  128비트, 16비트씩 8부분, 콜론(:) 구분, 16진수

  1계층(물리): 허브, 리피터
  2계층(데이터링크): 브리지, 스위치
  3계층(네트워크): 라우터
  전 계층: 게이트웨이

프로토콜 3요소: 구문(Syntax) + 의미(Semantics) + 시간(Timing)

```

### 보강 24: 기출에서 새로 등장한 용어

```
★ RTO (Recovery Time Objective): 업무 중단→복구까지 시간
  RPO (Recovery Point Objective): 복구 시 데이터 손실 허용 시점

★ SOAP: HTTP 통해 XML 메시지 교환 프로토콜

★ chmod 권한 설정 (8진법):
  읽기(r)=4, 쓰기(w)=2, 실행(x)=1
  chmod 751 a.txt
  → 소유자: 7(rwx), 그룹: 5(r-x), 기타: 1(--x)

★ Linked Open Data (LOD): Link Data + Open Data 합성어
★ 정적 분석 도구: 실행 없이 소스코드 분석 (코딩표준, 복잡도, 결함)
★ CREATE INDEX 인덱스명 ON 테이블명(속성명);
```
