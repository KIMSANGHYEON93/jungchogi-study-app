### 4-2. SQL 활용 (최중요)

#### 빈출 유형
- **SELECT 구문**: WHERE, GROUP BY, HAVING, ORDER BY
- **JOIN**: INNER JOIN, LEFT/RIGHT OUTER JOIN, CROSS JOIN
- **서브쿼리**: 스칼라, 인라인 뷰, 중첩
- **DDL**: CREATE, ALTER, DROP, TRUNCATE
- **DML**: INSERT, UPDATE, DELETE
- **DCL**: GRANT, REVOKE
- **집계함수**: COUNT, SUM, AVG, MAX, MIN

#### SQL 실행 순서 (필수 암기)
```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
```

#### 핵심 암기
```sql
-- JOIN 기본 형태
SELECT A.col, B.col
FROM 테이블A A
JOIN 테이블B B ON A.key = B.key
WHERE 조건
GROUP BY 그룹컬럼
HAVING 그룹조건
ORDER BY 정렬컬럼;

-- 윈도우 함수
SELECT col, RANK() OVER(PARTITION BY col ORDER BY col) FROM 테이블;
```

#### 프로시저 / 함수 / 트리거 (빈출)

```sql
-- 프로시저 기본 형태
CREATE OR REPLACE PROCEDURE 프로시저명(매개변수)
IS
  변수 선언;
BEGIN
  실행부;
EXCEPTION
  예외 처리;
END;

-- 트리거 기본 형태
CREATE TRIGGER 트리거명
BEFORE | AFTER  INSERT | UPDATE | DELETE
ON 테이블명
FOR EACH ROW
BEGIN
  실행부;  -- :NEW, :OLD 참조
END;

-- 사용자 정의 함수
CREATE FUNCTION 함수명(매개변수) RETURNS 데이터타입
BEGIN
  RETURN 값;
END;
```

### 4-4. UML (Unified Modeling Language)

#### UML 다이어그램 분류

**구조 다이어그램 (정적)**
- 클래스, 객체, 컴포넌트, 배치, 복합구조, 패키지

**행위 다이어그램 (동적)**
- 유스케이스, 시퀀스, 커뮤니케이션, 상태, 활동, 타이밍

#### UML 관계 (필수 암기)

| 관계 | 표기 | 설명 |
|------|------|------|
| 연관 (Association) | 실선 화살표 | 클래스 간 관계 |
| 의존 (Dependency) | 점선 화살표 | 일시적 사용 |
| 일반화 (Generalization) | 속이 빈 삼각형 | 상속 (is-a) |
| 실체화 (Realization) | 점선 + 빈 삼각형 | 인터페이스 구현 |
| 집합 (Aggregation) | 속이 빈 마름모 | 부분-전체 (독립적) |
| 합성 (Composition) | 속이 찬 마름모 | 부분-전체 (생명주기 공유) |

#### 결합도 / 응집도 (빈출)

```

(약) ──────────────────────────────→ (강)
```

```
우연 → 논리 → 시간 → 절차 → 통신 → 순차 → 기능
(약) ──────────────────────────────→ (강)
```

### 4-6. 데이터베이스 구축

#### 정규화 (필수)
| 정규형 | 조건 |
|--------|------|
| 1NF | 원자값 (도메인이 원자값) |
| 2NF | 부분 함수 종속 제거 |
| 3NF | 이행 함수 종속 제거 |
| BCNF | 결정자가 모두 후보키 |

> **암기법**: "원부이결다조" (원자 → 부분 → 이행 → 결정자 → 다치 → 조인)

#### 트랜잭션 ACID
- **A**tomicity (원자성): 전부 또는 전무
- **C**onsistency (일관성): 일관된 상태 유지
- **I**solation (격리성): 동시 실행 시 간섭 없음
- **D**urability (영속성): 완료 후 영구 반영

5. **모르는 문제도 공란 금지** - 아는 키워드라도 반드시 적기
6. **답안 작성 시 영어 대소문자 구분** - 키워드 정확히 작성

## 9. 핵심 요약: 합격을 위한 5가지 원칙

1. **코드 트레이싱이 생명이다** - 매일 C/Java/Python 기출 코드를 손으로 직접 따라가며 출력값을 예측하는 연습
2. **SQL은 외우는 게 아니라 쓰는 것이다** - JOIN, GROUP BY, 서브쿼리를 직접 작성하는 반복 연습
3. **기출이 답이다** - 최근 3개년(2023~2025) 기출을 최소 2회독, 오답 노트 필수
4. **약어/용어를 정확히 쓰자** - 채점 시 스펠링 오류로 감점, 영어 키워드 정확한 표기 연습
5. **시간 배분이 합불을 가른다** - 150분 안에 20문항, 코딩 문제에 시간 편중 방지
