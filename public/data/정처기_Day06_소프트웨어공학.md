## PART 1: 소프트웨어 개발 방법론

### 1-1. 전통적 방법론

| 모델 | 특징 | 장점 | 단점 |
|------|------|------|------|
| **폭포수** (Waterfall) | 순차적 진행, 되돌아감 없음 | 단순, 이해 쉬움 | 변경 어려움 |
| **프로토타입** | 시제품 먼저 개발 → 피드백 반영 | 요구사항 명확화 | 비용 증가 |
| **나선형** (Spiral) | 계획→**위험분석**→개발→평가 반복 | 위험 관리 우수 | 복잡, 비용 큼 |
| **V모델** | 폭포수 확장, 각 단계에 **테스트** 대응 | 테스트 체계적 | 반복 없음 |

> **시험 포인트**: **나선형** = "**위험 분석**"이 키워드!

### 1-2. 애자일 (Agile) 방법론

> 핵심 가치: **개인과 상호작용** > 프로세스/도구, **동작하는 소프트웨어** > 문서

#### 스크럼 (Scrum) ★★★★★

| 용어 | 설명 |
|------|------|
| **제품 백로그** (Product Backlog) | 전체 요구사항 목록 |
| **스프린트** (Sprint) | **2~4주** 짧은 개발 주기 |
| **스프린트 백로그** | 해당 스프린트에서 수행할 작업 목록 |
| **일일 스크럼** (Daily Scrum) | **매일 15분** 짧은 회의 (스탠드업) |
| **스크럼 마스터** | 프로세스 관리, 장애물 제거 역할 |
| **제품 책임자** (Product Owner) | 요구사항 우선순위 결정 |
| **번다운 차트** (Burn-down) | 남은 작업량 대비 시간 그래프 |
| **속도** (Velocity) | 스프린트당 처리 가능한 작업량 |

#### XP (eXtreme Programming) ★★★★★

**5가지 가치**: **용단의피존** (용기, 단순성, 의사소통, 피드백, 존중)

**12가지 실천 방법 (빈출!):**

| 실천 방법 | 설명 |
|-----------|------|
| **Pair Programming** (짝 프로그래밍) | 2명이 한 컴퓨터에서 코딩 |
| **TDD** (Test-Driven Development) | **테스트 먼저** 작성 후 코드 개발 |
| **Refactoring** (리팩토링) | 기능은 유지, 코드 구조 개선 |
| **CI** (Continuous Integration) | 지속적 통합, 자주 빌드/테스트 |
| **Small Release** | 짧은 주기로 릴리즈 |
| **Collective Ownership** | 코드 공동 소유 |
| **Planning Game** | 고객+개발자 함께 계획 |
| **Simple Design** | 가장 단순한 설계 |

> **시험 단골**: "테스트를 먼저 작성하고 이를 통과하는 코드를 개발하는 방법" → **TDD**

### 1-3. SOLID 원칙

| 원칙 | 영어 | 설명 |
|------|------|------|
| **S** | Single Responsibility | 클래스는 **하나의 책임**만 |
| **O** | Open-Closed | 확장에 열림, **수정에 닫힘** |
| **L** | Liskov Substitution | 자식은 부모를 **대체** 가능 |
| **I** | Interface Segregation | 인터페이스 **분리** |
| **D** | Dependency Inversion | 추상에 의존, 구체에 **비의존** |

## PART 2: 소프트웨어 테스트

### 2-1. 테스트 단계 (V모델 순서)

```
요구사항 분석  ←→  인수 테스트 (사용자 관점)
  설계         ←→  시스템 테스트 (전체 시스템)
    상세 설계   ←→  통합 테스트 (모듈 간 인터페이스)
      코딩      ←→  단위 테스트 (개별 모듈)
```

> **암기**: **단통시인** (단위→통합→시스템→인수)

### 2-2. 화이트박스 vs 블랙박스

| 구분 | 화이트박스 | 블랙박스 |
|------|-----------|---------|
| 관점 | 내부 로직 (코드 보임) | 외부 기능 (코드 안 보임) |
| 기준 | 코드 커버리지 | 명세/요구사항 |
| 주체 | 개발자 | 사용자/테스터 |

### 2-3. 화이트박스 테스트 기법 ★★★★

| 기법 | 설명 | 커버리지 강도 |
|------|------|-------------|
| **구문 커버리지** (Statement) | 모든 **문장** 최소 1번 실행 | 약 |
| **결정 커버리지** (Decision/Branch) | 모든 **분기** 참/거짓 실행 | ↓ |
| **조건 커버리지** (Condition) | 모든 개별 **조건** 참/거짓 | ↓ |
| **조건/결정 커버리지** | 조건+결정 모두 만족 | ↓ |
| **변경 조건/결정** (MC/DC) | 각 조건이 결정에 독립적 영향 | ↓ |
| **다중 조건 커버리지** | 모든 조건 **조합** 테스트 | 강 |

> **암기** (약→강): **구결조 조결변 다** (구문→결정→조건→조건/결정→변경조건/결정→다중조건)

### 2-4. 블랙박스 테스트 기법 ★★★★

| 기법 | 설명 |
|------|------|
| **동등분할** (Equivalence Partitioning) | 입력 범위를 **유효/무효 클래스**로 분류 |
| **경계값 분석** (Boundary Value) | 경계 부근 값을 테스트 |
| **결정 테이블** | 조건과 행동의 모든 조합 |
| **상태 전이** | 상태 변화에 따른 테스트 |
| **유스케이스** | 사용 시나리오 기반 테스트 |
| **원인-결과 그래프** | 입력(원인)과 출력(결과) 관계 |
| **오류 추정** | 경험 기반 오류 예측 |

### 2-5. 통합 테스트 ★★★★★

| 방식 | 방향 | 필요한 것 | 설명 |
|------|------|-----------|------|
| **하향식** (Top-down) | 상위 → 하위 | **스텁 (Stub)** | 하위 모듈 대체 가짜 모듈 |
| **상향식** (Bottom-up) | 하위 → 상위 | **드라이버 (Driver)** | 상위 모듈 대체 가짜 모듈 |
| **빅뱅** | 한 번에 전체 | - | 모든 모듈 동시 통합 |
| **샌드위치** | 양방향 | 스텁+드라이버 | 상향+하향 혼합 |

> **시험 핵심**: **하향식 = 스텁**, **상향식 = 드라이버**

### 2-6. 테스트 오라클

| 오라클 | 설명 |
|--------|------|
| **참 오라클** | **모든** 입력에 대한 기대 결과 알고 있음 |
| **샘플링 오라클** | **일부** 입력에 대해서만 기대 결과 |
| **휴리스틱 오라클** | **경험**에 기반한 결과 추정 |
| **일관성 검사 오라클** | 이전 실행 결과와 **비교** |

## PART 3: 정규화 (Normalization)

### 3-1. 정규화 단계

| 단계 | 조건 | 제거 대상 | 암기 |
|------|------|-----------|------|
| **1NF** | 모든 속성이 **원자값** | 반복 그룹 제거 | 도메인이 원자값 |
| **2NF** | 1NF + **부분 함수 종속** 제거 | 기본키 일부에 종속 | **부분** 제거 |
| **3NF** | 2NF + **이행 함수 종속** 제거 | A→B→C에서 A→C 제거 | **이행** 제거 |
| **BCNF** | 3NF + 모든 **결정자**가 후보키 | 결정자 but 후보키 아닌 것 제거 | **결정자** |
| **4NF** | BCNF + **다치 종속** 제거 | | |
| **5NF** | 4NF + **조인 종속** 제거 | | |

> **암기**: **도부이결다조** (도메인→부분→이행→결정자→다치→조인)
> 또는: "**두부이걸다죠?**" (두→도, 부→부분, 이→이행, 걸→결정, 다→다치, 죠→조인)

### 3-2. 반정규화 (De-normalization)

- 정규화된 테이블을 **의도적으로 중복** 허용
- **성능 향상**이 목적 (JOIN 줄이기)
- 데이터 무결성 저하 가능성

## PART 4: 네트워크

### 4-1. OSI 7계층 ★★★★★

| 계층 | 이름 | 역할 | 프로토콜/장비 |
|------|------|------|-------------|
| 7 | **응용** (Application) | 사용자 인터페이스 | HTTP, FTP, SMTP, DNS |
| 6 | **표현** (Presentation) | 데이터 형식 변환, 암호화 | JPEG, MPEG, SSL |
| 5 | **세션** (Session) | 연결 관리, 동기화 | RPC, NetBIOS |
| 4 | **전송** (Transport) | 신뢰성, 흐름 제어 | **TCP, UDP** |
| 3 | **네트워크** (Network) | **경로 설정** (라우팅) | IP, ICMP / **라우터** |
| 2 | **데이터링크** (Data Link) | **MAC 주소**, 오류 제어 | Ethernet / **스위치**, 브리지 |
| 1 | **물리** (Physical) | 전기 신호 전송 | **허브**, 리피터 |

> **암기**: **아(응)파(표)서(세)전(전)네(네)데(데)피(물)** 또는 **물데네전세표응** (아래→위)

### 4-2. TCP vs UDP

| 구분 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결형 (3-way handshake) | 비연결형 |
| 신뢰성 | 높음 (순서 보장, 재전송) | 낮음 |
| 속도 | 느림 | 빠름 |
| 용도 | HTTP, FTP, 이메일 | DNS, 스트리밍, VoIP |

### 4-3. 주요 프로토콜

| 프로토콜 | 포트 | 설명 |
|----------|------|------|
| **HTTP** | 80 | 웹 페이지 전송 |
| **HTTPS** | 443 | HTTP + SSL/TLS 암호화 |
| **FTP** | 20/21 | 파일 전송 |
| **SSH** | 22 | 원격 접속 (암호화) |
| **TELNET** | 23 | 원격 접속 (평문) |
| **SMTP** | 25 | 메일 송신 |
| **POP3** | 110 | 메일 수신 |
| **DNS** | 53 | 도메인→IP 변환 |
| **DHCP** | 67/68 | IP 자동 할당 |
| **SNMP** | 161 | 네트워크 장비 관리 |

### 4-4. IP 주소

| 클래스 | 범위 | 기본 서브넷 |
|--------|------|------------|
| A | 0~127 | 255.0.0.0 |
| B | 128~191 | 255.255.0.0 |
| C | 192~223 | 255.255.255.0 |

**IPv4 vs IPv6:**

| 구분 | IPv4 | IPv6 |
|------|------|------|
| 주소 길이 | **32비트** | **128비트** |
| 표현 | 10진수 (192.168.0.1) | 16진수 (2001:0db8::) |
| 주소 수 | 약 43억 | 사실상 무한 |

> **전환 기술**: **듀얼 스택**, **터널링**, **헤더 변환**

## PART 5: 보안

### 5-1. 보안 3대 요소 (CIA)

| 요소 | 영어 | 설명 |
|------|------|------|
| **기밀성** | Confidentiality | 인가된 사용자만 접근 |
| **무결성** | Integrity | 데이터 변조 방지 |
| **가용성** | Availability | 필요할 때 사용 가능 |

### 5-2. 암호화 방식

| 구분 | 대칭키 (비밀키) | 비대칭키 (공개키) |
|------|-----------------|------------------|
| 키 | 암복호화 키 **동일** | 공개키+개인키 **한 쌍** |
| 속도 | 빠름 | 느림 |
| 알고리즘 | **DES, 3DES, AES, SEED, ARIA** | **RSA, ECC, DSA** |
| 키 관리 | 어려움 (n명이면 n(n-1)/2개) | 쉬움 (2n개) |

> **시험 포인트**: **AES** = 대칭키, **RSA** = 비대칭키 (가장 빈출!)

### 5-3. 해시 함수

- 일방향 함수 (복호화 불가)
- 종류: **MD5**, **SHA-1**, **SHA-256**, **SHA-512**
- 용도: 비밀번호 저장, 무결성 검증

### 5-4. 서비스 공격 유형 ★★★★★

| 공격 | 설명 |
|------|------|
| **DoS** | 한 대의 공격자가 서버 자원 고갈 |
| **DDoS** | **여러 대**의 분산된 공격자가 동시 공격 |
| **Land Attack** | 출발지=목적지 주소 동일하게 위조 |
| **Smurf Attack** | ICMP Echo를 **브로드캐스트**로 대량 전송 |
| **SYN Flooding** | TCP 3-way 연결 요청만 반복 (ACK 안보냄) |
| **Ping of Death** | 비정상적으로 큰 ICMP 패킷 전송 |
| **Teardrop** | 패킷 오프셋을 조작하여 재조립 방해 |

### 5-5. 네트워크/웹 공격

| 공격 | 설명 |
|------|------|
| **스니핑** (Sniffing) | 네트워크 패킷을 **몰래 엿봄** (수동적) |
| **스푸핑** (Spoofing) | IP/MAC 주소를 **위조** |
| **SQL Injection** | 입력값에 **SQL문 삽입**하여 DB 조작 |
| **XSS** (Cross-Site Scripting) | 웹페이지에 **악성 스크립트 삽입** |
| **CSRF** | 사용자 의지와 무관하게 **요청 위조** |
| **세션 하이재킹** | 세션 ID를 탈취하여 **세션 가로챔** |
| **브루트포스** (Brute Force) | 모든 조합을 시도하는 **무차별 대입** |
| **사전 공격** (Dictionary) | 자주 쓰는 단어 목록으로 비밀번호 추측 |

### 5-6. 악성코드 / 고급 공격

| 용어 | 설명 |
|------|------|
| **랜섬웨어** (Ransomware) | 파일 **암호화** 후 금전 요구 |
| **APT** (Advanced Persistent Threat) | **지속적이고 지능적인** 맞춤형 공격 |
| **제로데이** (Zero-Day) | 패치 전 **알려지지 않은** 취약점 공격 |
| **트로이 목마** (Trojan Horse) | 정상 프로그램으로 **위장** |
| **웜** (Worm) | 스스로 **자가 복제**하여 전파 |
| **백도어** (Backdoor) | 정상 인증을 우회하는 **비밀 통로** |
| **키로거** (Keylogger) | **키보드 입력** 정보 탈취 |
| **루트킷** (Rootkit) | 시스템 관리자 권한 탈취, 자신을 은폐 |

### 5-7. 보안 솔루션

| 솔루션 | 설명 |
|--------|------|
| **방화벽** (Firewall) | 내외부 네트워크 경계에서 접근 제어 |
| **IDS** (Intrusion Detection) | 침입 **탐지** 시스템 |
| **IPS** (Intrusion Prevention) | 침입 **방지** 시스템 (탐지+차단) |
| **VPN** | 가상 사설 네트워크 (암호화 터널) |
| **WAF** | 웹 애플리케이션 방화벽 |
| **DLP** | 데이터 유출 방지 |
| **NAC** | 네트워크 접근 제어 |
| **허니팟** (Honeypot) | 해커를 유인하는 **가짜 시스템** |

### 5-8. 접근 제어

| 방식 | 영어 | 설명 |
|------|------|------|
| **DAC** | Discretionary | 소유자가 **임의로** 권한 부여 |
| **MAC** | Mandatory | **시스템 정책**으로 권한 결정 (군사용) |
| **RBAC** | Role-Based | **역할** 기반 접근 제어 |

## PART 6: 운영체제 핵심

### 6-1. 교착상태 (Deadlock) ★★★★

**발생 조건 4가지** (모두 충족 시 교착상태):

| 조건 | 영어 | 설명 |
|------|------|------|
| **상호 배제** | Mutual Exclusion | 하나의 프로세스만 자원 사용 |
| **점유와 대기** | Hold & Wait | 자원 보유한 채 다른 자원 대기 |
| **비선점** | Non-preemption | 강제로 자원 빼앗을 수 없음 |
| **환형 대기** | Circular Wait | 프로세스들이 원형으로 대기 |

> **암기**: **상점비환** (상호배제, 점유대기, 비선점, 환형대기)

**해결 방법:**

| 방법 | 설명 |
|------|------|
| **예방** (Prevention) | 4개 조건 중 하나 제거 |
| **회피** (Avoidance) | **은행원 알고리즘** (안전 상태 유지) |
| **탐지** (Detection) | 교착상태 발생 후 감지 |
| **회복** (Recovery) | 교착상태 해소 (프로세스 강제 종료 등) |

### 6-2. 스케줄링

| 방식 | 선점 여부 | 설명 |
|------|-----------|------|
| **FCFS** (FIFO) | 비선점 | 먼저 온 순서대로 |
| **SJF** | 비선점 | 실행 시간 짧은 것 먼저 |
| **SRT** | **선점** | 남은 시간 짧은 것 먼저 (SJF 선점 버전) |
| **RR** (Round Robin) | **선점** | 시간 할당량(Time Quantum)만큼 돌아가며 |
| **Priority** | 둘 다 가능 | 우선순위 높은 것 먼저 |
| **HRN** | 비선점 | (대기시간+서비스시간)/서비스시간 |

> **선점 스케줄링**: **SRT, RR** (나머지는 비선점)
