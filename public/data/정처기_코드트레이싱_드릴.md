### C-01. 포인터 기본

```c
#include <stdio.h>
int main() {
    int a = 10, b = 20;
    int *p = &a;
    *p = *p + b;
    p = &b;
    *p = *p + a;
    printf("%d %d", a, b);
    return 0;
}
```

<details>
<summary>정답 및 풀이</summary>

```
추적표:
a=10, b=20, p=&a
*p = *p + b → *p = 10+20 = 30 → a=30
p = &b → p가 b를 가리킴
*p = *p + a → *p = 20+30 = 50 → b=50

출력: 30 50
```
**함정**: `*p + b`에서 a가 30으로 바뀐 뒤, 이후 `*p + a`의 a는 이미 30임
</details>

### C-02. 후위/전위 증감

```c
#include <stdio.h>
int main() {
    int a = 5, b = 5;
    int c = a++ + ++b;
    int d = ++a + b++;
    printf("%d %d %d %d", a, b, c, d);
    return 0;
}
```

<details>
<summary>정답 및 풀이</summary>

```
a=5, b=5
c = a++ + ++b
  = 5(사용후a증가) + 6(증가후사용) = 11 → a=6, b=6, c=11
d = ++a + b++
  = 7(증가후사용) + 6(사용후b증가) = 13 → a=7, b=7, d=13

출력: 7 7 11 13
```
**함정**: a++는 현재값 사용 후 증가, ++b는 증가 후 사용. 순서 헷갈리지 말 것
</details>

### C-03. 배열과 포인터

```c
#include <stdio.h>
int main() {
    int a[] = {1, 2, 3, 4, 5};
    int *p = a + 1;
    printf("%d ", *p);
    printf("%d ", *(p + 2));
    printf("%d ", *p + 2);
    return 0;
}
```

<details>
<summary>정답 및 풀이</summary>

```
a = {1,2,3,4,5}
p = a+1 → p는 a[1] 가리킴

*p       → a[1] = 2
*(p+2)   → a[3] = 4
*p + 2   → a[1]+2 = 2+2 = 4

출력: 2 4 4
```
**핵심 함정**: `*(p+2)` = 주소이동 후 역참조 vs `*p+2` = 역참조 후 +2. 결과가 같을 수 있으나 원리가 다름!
</details>

### C-04. 재귀함수

```c
#include <stdio.h>
int f(int n) {
    if (n <= 1) return 1;
    return n * f(n - 2);
}
int main() {
    printf("%d", f(7));
    return 0;
}
```

<details>
<summary>정답 및 풀이</summary>

```
f(7) = 7 * f(5)
f(5) = 5 * f(3)
f(3) = 3 * f(1)
f(1) = 1 (종료조건)

역추적: f(3)=3*1=3, f(5)=5*3=15, f(7)=7*15=105

출력: 105
```
**함정**: n-2씩 감소 (n-1이 아님). 종료조건 n<=1 확인
</details>

### C-07. swap 함수 (주소 전달)

```c
#include <stdio.h>
void swap(int *a, int *b) {
    int t = *a;
    *a = *b;
    *b = t;
}
int main() {
    int x = 3, y = 7;
    swap(&x, &y);
    printf("%d %d\n", x, y);

    int *p = &x, *q = &y;
    int temp = *p;
    *p = *q;
    *q = temp;
    printf("%d %d", x, y);
    return 0;
}
```

<details>
<summary>정답 및 풀이</summary>

```
x=3, y=7
swap(&x, &y) → x=7, y=3 (주소전달이므로 실제 교환됨)
printf: 7 3

p=&x(7), q=&y(3)
temp=*p=7, *p=*q=3, *q=temp=7 → x=3, y=7
printf: 3 7

출력:
7 3
3 7
```
**함정**: 두 번 swap하면 원래대로 돌아옴
</details>

### C-10. 비트 연산

```c
#include <stdio.h>
int main() {
    int a = 12;     // 1100
    int b = 10;     // 1010
    printf("%d ", a & b);
    printf("%d ", a | b);
    printf("%d ", a ^ b);
    printf("%d", a >> 2);
    return 0;
}
```

<details>
<summary>정답 및 풀이</summary>

```
a = 12 = 1100
b = 10 = 1010

a & b = 1000 = 8  (AND: 둘 다 1이면 1)
a | b = 1110 = 14 (OR: 하나라도 1이면 1)
a ^ b = 0110 = 6  (XOR: 다르면 1)
a >> 2 = 0011 = 3 (오른쪽 2칸 = /4)

출력: 8 14 6 3
```
**암기**: `>>n = /2^n`, `<<n = *2^n`
</details>

### J-10. 싱글톤 패턴

```java
class Singleton {
    private static Singleton instance;
    private int value = 0;
    private Singleton() {}
    public static Singleton getInstance() {
        if (instance == null) instance = new Singleton();
        return instance;
    }
    public void add(int n) { value += n; }
    public int getValue() { return value; }
}
public class Main {
    public static void main(String[] args) {
        Singleton a = Singleton.getInstance();
        Singleton b = Singleton.getInstance();
        a.add(5);
        b.add(3);
        System.out.println(a.getValue() + " " + b.getValue());
    }
}
```

<details>
<summary>정답 및 풀이</summary>

```
a = getInstance() → 새 인스턴스 생성 (value=0)
b = getInstance() → 이미 존재 → 같은 인스턴스 반환
a와 b는 동일 객체!

a.add(5) → value = 0+5 = 5
b.add(3) → value = 5+3 = 8 (같은 객체이므로)

출력: 8 8
```
**함정**: 싱글톤은 하나의 인스턴스만 존재. a와 b는 같은 객체를 참조
</details>

### S-01. GROUP BY + HAVING

```
테이블: 사원(이름, 부서, 급여)
| 이름 | 부서 | 급여 |
|------|------|------|
| 김 | 개발 | 400 |
| 이 | 개발 | 300 |
| 박 | 인사 | 350 |
| 정 | 개발 | 500 |
| 최 | 인사 | 250 |
```

```sql
SELECT 부서, COUNT(*) AS 인원, AVG(급여) AS 평균
FROM 사원
GROUP BY 부서
HAVING COUNT(*) >= 3;
```

<details>
<summary>정답 및 풀이</summary>

```
① GROUP BY 부서:
   개발: 김(400), 이(300), 정(500) → 3명
   인사: 박(350), 최(250) → 2명

② HAVING COUNT(*) >= 3:
   개발: 3 >= 3 → O
   인사: 2 >= 3 → X (제외)

결과:
| 부서 | 인원 | 평균 |
|------|------|------|
| 개발 | 3 | 400 |
```
**함정**: AVG(급여) = (400+300+500)/3 = 400. HAVING은 그룹 후 필터
</details>

### S-02. JOIN

```
테이블: 학생(학번, 이름), 수강(학번, 과목)
학생:                  수강:
| 학번 | 이름 |       | 학번 | 과목 |
|------|------|       |------|------|
| 1 | 김 |           | 1 | DB |
| 2 | 이 |           | 1 | OS |
| 3 | 박 |           | 2 | DB |
                      | 4 | NET |
```

```sql
SELECT 학생.이름, 수강.과목
FROM 학생 LEFT JOIN 수강 ON 학생.학번 = 수강.학번;
```

<details>
<summary>정답 및 풀이</summary>

```
LEFT JOIN = 왼쪽(학생) 전체 + 오른쪽 매칭

학번1 김 → DB, OS (2행)
학번2 이 → DB (1행)
학번3 박 → 매칭 없음 → NULL

결과:
| 이름 | 과목 |
|------|------|
| 김 | DB |
| 김 | OS |
| 이 | DB |
| 박 | NULL |
```
**핵심**: LEFT JOIN = 왼쪽 테이블 전부 포함, 매칭 없으면 NULL. 수강의 학번4는 학생에 없으므로 미포함
</details>

### S-04. COUNT(*) vs COUNT(컬럼)

```
테이블: 주문(주문번호, 고객명, 할인코드)
| 주문번호 | 고객명 | 할인코드 |
|----------|--------|----------|
| 1 | 김 | A01 |
| 2 | 이 | NULL |
| 3 | 박 | B02 |
| 4 | 정 | NULL |
| 5 | 최 | A01 |
```

```sql
SELECT COUNT(*), COUNT(할인코드), COUNT(DISTINCT 할인코드)

```

<details>
<summary>정답 및 풀이</summary>

```

```
**필수 암기**: COUNT(*) = NULL 포함, COUNT(컬럼) = NULL 제외

### S-08. GRANT / REVOKE

    ___ SELECT, INSERT ON 학생 TO kim ___ ___ ___;

### S-09. INNER JOIN + GROUP BY

```
테이블: 고객(고객ID, 이름), 주문(주문ID, 고객ID, 금액)
고객:                    주문:
| 고객ID | 이름 |        | 주문ID | 고객ID | 금액 |
|--------|------|        |--------|--------|------|
| 1 | 김 |              | 101 | 1 | 5000 |
| 2 | 이 |              | 102 | 1 | 3000 |
| 3 | 박 |              | 103 | 2 | 7000 |
                         | 104 | 4 | 2000 |
```

```sql
SELECT 고객.이름, SUM(주문.금액) AS 총액
FROM 고객 INNER JOIN 주문 ON 고객.고객ID = 주문.고객ID
GROUP BY 고객.이름
HAVING SUM(주문.금액) >= 5000;
```

<details>
<summary>정답 및 풀이</summary>

```
① INNER JOIN (양쪽 일치만):
   고객1 김 ↔ 주문101(5000), 주문102(3000)
   고객2 이 ↔ 주문103(7000)
   고객3 박 → 매칭 없음 (제외)
   주문104 고객4 → 고객에 없음 (제외)

② GROUP BY 이름:
   김: 5000+3000 = 8000
   이: 7000

③ HAVING SUM >= 5000:
   김: 8000 >= 5000 → O
   이: 7000 >= 5000 → O

결과:
| 이름 | 총액 |
|------|------|
| 김 | 8000 |
| 이 | 7000 |
```
**핵심**: INNER JOIN = 양쪽 모두 있는 것만. 박(주문없음)과 주문104(고객없음) 모두 제외
</details>

### S-10. IS NULL + COALESCE

```
테이블: 사원(사번, 이름, 관리자사번)
| 사번 | 이름 | 관리자사번 |
|------|------|------------|
| 1 | 김 | NULL |
| 2 | 이 | 1 |
| 3 | 박 | 1 |
| 4 | 정 | NULL |
| 5 | 최 | 2 |
```

```sql
SELECT 이름,
       CASE WHEN 관리자사번 IS NULL THEN '대표'
            ELSE '직원' END AS 직급
FROM 사원
WHERE 관리자사번 IS NULL OR 관리자사번 = 1;
```

<details>
<summary>정답 및 풀이</summary>

```
WHERE 필터:
  사번1 김: NULL IS NULL → O
  사번2 이: 관리자=1 → O
  사번3 박: 관리자=1 → O
  사번4 정: NULL IS NULL → O
  사번5 최: 관리자=2, NULL아님 → X

CASE 적용:
  김: NULL → '대표'
  이: 1 → '직원'
  박: 1 → '직원'
  정: NULL → '대표'

결과:
| 이름 | 직급 |
|------|------|
| 김 | 대표 |
| 이 | 직원 |
| 박 | 직원 |
| 정 | 대표 |
```
**필수**: NULL 비교는 반드시 `IS NULL` 사용. `= NULL`은 틀림!
</details>

## Part 5. 함정 유형 TOP 10 총정리

```
┌──────────────────────────────────────────────────────┐
│  ★ 시험장에서 가장 많이 틀리는 함정 10가지 ★        │
├──────────────────────────────────────────────────────┤
│                                                      │
│  1. a++ vs ++a                                       │
│     → a++ = 현재값 사용 후 증가                      │
│     → ++a = 증가 후 사용                             │
│                                                      │
│  2. *p+2 vs *(p+2)                                   │
│     → *p+2 = 값에 +2                                 │
│     → *(p+2) = 주소 2칸 이동 후 역참조               │
│                                                      │
│  3. 변수=부모, 메서드=자식 (Java 다형성)             │
│     → A obj = new B();                               │
│     → obj.변수 = A것, obj.메서드() = B것             │
│                                                      │
│  4. == vs equals() (Java String)                     │
│     → == 참조비교, equals() 값비교                   │
│     → 리터럴끼리는 == 도 true (Pool 공유)            │
│                                                      │
│  5. substring(s,e) / 슬라이싱[s:e]                   │
│     → e는 미포함! (s부터 e-1까지)                    │
│                                                      │
│  6. range(n) / range(s,e)                            │
│     → range(5) = 0,1,2,3,4 (0부터, n 미포함)        │
│                                                      │
│  7. COUNT(*) vs COUNT(컬럼)                          │
│     → COUNT(*) = NULL 포함                           │
│     → COUNT(컬럼) = NULL 제외                        │
│                                                      │
│  8. WHERE vs HAVING                                  │
│     → WHERE = 그룹 전 필터 (집계함수 X)              │
│     → HAVING = 그룹 후 필터 (집계함수 O)             │
│                                                      │
│  9. > vs >= (초과 vs 이상)                           │
│     → WHERE 급여 > 400 → 400 미포함!                 │
│     → WHERE 급여 >= 400 → 400 포함                   │
│                                                      │
│  10. = NULL (X) → IS NULL (O)                        │
│      → NULL 비교는 반드시 IS NULL / IS NOT NULL       │
│                                                      │
└──────────────────────────────────────────────────────┘
```
