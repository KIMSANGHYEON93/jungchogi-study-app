## SECTION 1: 소프트웨어 설계

### 1-1. 요구사항

| 용어 | 설명 |
|------|------|
| **기능적 요구사항** | 시스템이 **무엇을** 해야 하는지 (입출력, DB, 통신) |
| **비기능적 요구사항** | **어떻게** 수행되어야 하는지 (성능, 보안, 품질, 안정) |
| **요구사항 개발** | **도분명확** (도출→분석→명세→확인) |
| **요구사항 도출** | 인터뷰, 설문, 브레인스토밍, 워크숍, 프로토타이핑 |
| **요구사항 명세** | 정형 명세 (수학적) / 비정형 명세 (자연어) |
| **요구사항 확인** | 리뷰, 워크스루, 인스펙션, 프로토타이핑 |

### 1-2. 화면 설계

| 용어 | 설명 |
|------|------|
| **와이어프레임** | 페이지의 **레이아웃/뼈대** (선과 도형으로 표현) |
| **스토리보드** | 와이어프레임 + **설명 + 페이지 이동 흐름** |
| **프로토타입** | **실제 작동하는** 시제품 모형 |
| **목업** (Mockup) | **실물과 유사한** 정적 모형 (디자인 중심) |

### 1-3. 객체지향 분석 — 럼바우 ★★★★★

| 모델링 | 다이어그램 | 설명 |
|--------|-----------|------|
| **객체** 모델링 | 객체 다이어그램 | 클래스 속성/관계 정의 |
| **동적** 모델링 | **상태** 다이어그램 | 상태 변화, 이벤트 흐름 |
| **기능** 모델링 | **DFD** (자료흐름도) | 프로세스 간 데이터 흐름 |

> **암기**: **럼바우 = 객동기** (객체→동적→기능)

### 1-4. 분석 도구

| 용어 | 설명 |
|------|------|
| **DFD** (Data Flow Diagram) | 데이터 흐름도: 프로세스(○), 데이터흐름(→), 저장소(=), 외부(□) |
| **DD** (Data Dictionary) | 자료사전: `=`정의, `+`연결, `{}`반복, `[]`선택, `()`선택사항 |
| **HIPO** | 기능 중심 시스템 문서화 (가시적 차트, 총체적 차트, 세부적 차트) |
| **CASE** | 소프트웨어 개발 자동화 도구 (상위/하위/통합 CASE) |

### 1-5. 디자인 패턴 분류 (복습)

```
생성(5): 빌프팩앱싱 (Builder, Prototype, Factory Method, Abstract Factory, Singleton)
```

| 키 | 설명 |
|----|------|
| **슈퍼키** | 유일성 O, 최소성 X |

### 3-3. 정규화 (복습)

```
도부이결다조
1NF(원자값) → 2NF(부분종속제거) → 3NF(이행종속제거) → BCNF(결정자) → 4NF(다치) → 5NF(조인)
```

### 3-4. 트랜잭션 ACID (복습)

```
원자성(Atomicity) — 전부 or 전무
일관성(Consistency) — 상태 일관 유지
격리성(Isolation) — 동시 실행 간섭 없음
영속성(Durability) — 완료 결과 영구 보존
```

### 3-5. 이상 (Anomaly)

| 이상 | 설명 |
|------|------|

### 5-1. 소프트웨어 품질 표준

| 표준 | 설명 |
|------|------|
| **ISO/IEC 9126** | 품질 특성 6가지: **기신사효유이** |
| **ISO/IEC 25010** | 9126 개정판, 8가지 품질 특성 |

**ISO 9126 — 기신사효유이:**

| 특성 | 설명 |
|------|------|
| **기능성** (Functionality) | 요구 기능을 제공하는가 |
| **신뢰성** (Reliability) | 성능 수준을 유지하는가 |
| **사용성** (Usability) | 쉽게 사용할 수 있는가 |
| **효율성** (Efficiency) | 자원 대비 성능이 적절한가 |
| **유지보수성** (Maintainability) | 수정이 용이한가 |
| **이식성** (Portability) | 다른 환경에서 사용 가능한가 |

### 5-2. 프로세스 성숙도 모델

**CMM / CMMI 5단계:**

| 단계 | 이름 | 설명 |
|------|------|------|
| 1 | **초기** (Initial) | 정의된 프로세스 없음 |
| 2 | **관리** (Managed) | 프로젝트 단위 관리 |
| 3 | **정의** (Defined) | 조직 차원 표준 프로세스 |
| 4 | **정량적 관리** (Quantitatively) | 정량적 측정/분석 |
| 5 | **최적화** (Optimizing) | 지속적 프로세스 개선 |

> **암기**: **초관정정최** (초기→관리→정의→정량적관리→최적화)

**SPICE (ISO 15504) 6단계:**

| 단계 | 이름 |
|------|------|
| 0 | 불완전 (Incomplete) |
| 1 | 수행 (Performed) |
| 2 | 관리 (Managed) |
| 3 | 확립 (Established) |
| 4 | 예측 (Predictable) |
| 5 | 최적화 (Optimizing) |
