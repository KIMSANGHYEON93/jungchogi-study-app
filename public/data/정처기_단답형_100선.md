## A. 데이터베이스 (최빈출 영역 1위) — 25문제

### 001. 트랜잭션의 4가지 특성(ACID)을 쓰시오.

<details><summary>정답</summary>

- **A**tomicity (원자성): 전부 실행 또는 전부 취소
- **C**onsistency (일관성): 실행 후 일관된 상태 유지
- **I**solation (독립성/고립성): 동시 실행 시 상호 간섭 불가
- **D**urability (지속성/영속성): 완료된 결과 영구 반영

</details>

### 002. 정규화 단계별 핵심을 쓰시오: 1NF, 2NF, 3NF, BCNF

<details><summary>정답</summary>

- **1NF**: 모든 속성이 **원자값** (더 이상 분해 불가)
- **2NF**: **부분 함수 종속** 제거
- **3NF**: **이행 함수 종속** 제거 (A→B→C에서 A→C 제거)
- **BCNF**: 모든 **결정자**가 후보키
- 암기: **도부이결다조**

</details>

### 003. 다음 키의 정의를 쓰시오: 슈퍼키, 후보키, 기본키, 외래키

<details><summary>정답</summary>

- **슈퍼키**: 유일성 O, 최소성 X
- **후보키**: 유일성 O, 최소성 O (튜플을 유일하게 식별하는 최소 속성 집합)
- **기본키**: 후보키 중 선정, NULL 불가, 중복 불가
- **외래키**: 다른 릴레이션의 기본키를 참조하는 속성

</details>

### 004. 이상(Anomaly)의 3가지 종류를 쓰시오.

<details><summary>정답</summary>

- **삽입 이상**: 원하지 않는 데이터까지 삽입해야 하는 현상
- **삭제 이상**: 의도와 상관없는 데이터까지 삭제되는 현상
- **갱신 이상**: 일부만 갱신되어 데이터 불일치가 발생하는 현상

</details>

### 005. 릴레이션에서 행의 수를 ( ① ), 열의 수를 ( ② )라 한다.

<details><summary>정답</summary>

① **카디널리티** (Cardinality, 기수, 대응수)
② **디그리** (Degree, 차수)

</details>

### 006. DELETE, TRUNCATE, DROP의 차이를 쓰시오.

<details><summary>정답</summary>

| | DELETE | TRUNCATE | DROP |
|--|--------|----------|------|
| 분류 | DML | DDL | DDL |
| 구조 | 유지 | 유지 | **삭제** |
| 롤백 | **가능** | 불가 | 불가 |
| WHERE | 사용 가능 | 불가 | 불가 |

</details>

### 007. SQL 실행 순서를 쓰시오.

<details><summary>정답</summary>

**FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY**

</details>

### 008. WHERE와 HAVING의 차이를 쓰시오.

<details><summary>정답</summary>

- **WHERE**: 그룹화 **전** 필터링, 집계함수 사용 불가
- **HAVING**: 그룹화 **후** 필터링, 집계함수 사용 가능

</details>

### 009. GRANT와 REVOKE 문법을 쓰시오.

<details><summary>정답</summary>

```sql
GRANT 권한 ON 테이블 TO 사용자 [WITH GRANT OPTION];
REVOKE 권한 ON 테이블 FROM 사용자 [CASCADE];
```
- **WITH GRANT OPTION**: 받은 권한을 다른 사용자에게 부여 가능
- **CASCADE**: 연쇄적으로 권한 취소

</details>

### 010. 인덱스를 생성하는 SQL문을 쓰시오. (테이블: student, 속성: name, 인덱스명: idx_name)

<details><summary>정답</summary>

```sql
CREATE INDEX idx_name ON student(name);
```

</details>

### 011. 뷰(View)란 무엇인지 쓰시오.

<details><summary>정답</summary>

하나 이상의 기본 테이블로부터 유도된 **가상 테이블**. 물리적으로 존재하지 않으며, 논리적 독립성을 제공한다. CREATE VIEW로 생성, DROP VIEW로 제거.

</details>

### 012. 반정규화(Denormalization)란?

<details><summary>정답</summary>

**성능 향상**을 위해 정규화된 데이터 모델을 의도적으로 통합, 중복, 분리하는 과정. 방법: 테이블 통합, 테이블 분할(수평/수직), 중복 테이블 추가, 중복 속성 추가.

</details>

### 013. 프로시저(Procedure)와 트리거(Trigger)의 차이를 쓰시오.

<details><summary>정답</summary>

- **프로시저**: **호출**에 의해 실행되는 절차형 SQL
- **트리거**: 데이터의 삽입/갱신/삭제 **이벤트 발생 시 자동** 실행되는 절차형 SQL

</details>

### 014. 데이터베이스 설계 순서를 쓰시오.

<details><summary>정답</summary>

요구사항 분석 → **개념적 설계** → **논리적 설계** → **물리적 설계**

</details>

### 015. 무결성의 종류 중 "기본키는 NULL이나 중복값을 가질 수 없다"는?

<details><summary>정답</summary>

**개체 무결성** (Entity Integrity)

- 참조 무결성: 외래키 = NULL 또는 참조 테이블 기본키
- 도메인 무결성: 속성값이 정의된 도메인 범위 내

</details>

### 016. ★2024 신유형: 다음 JOIN 유형을 설명하시오: 세미 조인, 세타 조인

<details><summary>정답</summary>

- **세미 조인(Semi Join)**: 한쪽 릴레이션의 조건 만족 튜플만 반환 (다른쪽 속성은 미포함)
- **세타 조인(Theta Join)**: 비교 연산자(=, <, >, ≤, ≥, ≠)를 사용한 조인

</details>

### 017. 데이터 모델의 구성 요소 3가지를 쓰시오.

<details><summary>정답</summary>

- **개체** (Entity)
- **속성** (Attribute)
- **관계** (Relationship)

</details>

### 018. 데이터 모델에 표시할 요소 3가지를 쓰시오.

<details><summary>정답</summary>

- **구조** (Structure): 개체 타입 간의 관계, 정적 성질
- **연산** (Operation): 데이터 조작 작업 명세
- **제약조건** (Constraint): 데이터의 논리적 제약

</details>

### 019. 릴레이션에서 하나의 속성이 취할 수 있는 같은 타입의 원자값 집합을 무엇이라 하는가?

<details><summary>정답</summary>

**도메인** (Domain)

</details>

### 020. 함수적 종속에서 X→Y일 때, X를 ( ① ), Y를 ( ② )라 한다.

<details><summary>정답</summary>

① **결정자** (Determinant)
② **종속자** (Dependent)

</details>

### 021. 고급 데이터베이스 용어를 설명하시오: 데이터 웨어하우스, 데이터 마이닝, OLAP

<details><summary>정답</summary>

- **데이터 웨어하우스**: 대량 데이터를 효과적으로 분석·정보화한 데이터베이스
- **데이터 마이닝**: 데이터에서 유용한 정보/패턴을 발견하는 기법
- **OLAP**: 다차원 데이터의 통계적 분석으로 의사결정 지원

</details>

### 022. COUNT(*)와 COUNT(컬럼)의 차이를 쓰시오.

<details><summary>정답</summary>

- **COUNT(*)**: NULL **포함** 전체 행 수
- **COUNT(컬럼)**: NULL **제외** 행 수

</details>

### 023. COMMIT, ROLLBACK, SAVEPOINT를 설명하시오.

<details><summary>정답</summary>

- **COMMIT**: 트랜잭션 변경 내용을 DB에 **영구 반영**
- **ROLLBACK**: 변경 내용을 **취소**하고 이전 상태로 복구
- **SAVEPOINT**: 트랜잭션 내 **저장점** 지정 (ROLLBACK TO로 해당 지점까지 취소)

</details>

### 024. CASCADE와 RESTRICT의 차이를 쓰시오.

<details><summary>정답</summary>

- **CASCADE**: 참조하는 모든 개체를 **함께 제거/변경**
- **RESTRICT**: 참조하는 개체가 있으면 **제거 취소**

</details>

### 025. ★DBMS의 3가지 필수 기능을 쓰시오.

<details><summary>정답</summary>

- **정의 기능** (Definition): 데이터 구조, 타입, 제약조건 명시
- **조작 기능** (Manipulation): 검색, 갱신, 삽입, 삭제
- **제어 기능** (Control): 무결성 유지, 접근 통제, 동시성 제어

</details>

<details><summary>정답</summary>

<details><summary>정답</summary>

<details><summary>정답</summary>

- 우연적 응집도가 가장 나쁘고, 기능적 응집도가 가장 좋음

### 033. 팬인(Fan-In)과 팬아웃(Fan-Out)을 설명하시오.

<details><summary>정답</summary>

- **팬인**: 해당 모듈을 **호출하는** 모듈의 수 (높을수록 재사용성 좋음)
- **팬아웃**: 해당 모듈이 **호출하는** 모듈의 수

</details>

### 034. 비용 산정 기법 중 LOC 기법의 공식을 쓰시오.

<details><summary>정답</summary>

- 노력(인월) = LOC / 1인당 월평균 생산 코드 라인 수
- 개발 비용 = 노력 × 단위 비용
- 개발 기간 = 노력 / 투입 인원
- 생산성 = LOC / 노력

</details>

</details>

### 053. "객체의 상태를 저장하고 복구(Undo)"하는 패턴은?

<details><summary>정답</summary>

**Memento** (메멘토) — 행위 패턴

</details>

### 054. 생성 패턴 5개를 쓰시오.

<details><summary>정답</summary>

**Builder, Prototype, Factory Method, Abstract Factory, Singleton** (빌프팩앱싱)

</details>

### 055. UML 구조(정적) 다이어그램 6개를 쓰시오.

<details><summary>정답</summary>

클래스, 객체, 컴포넌트, 배치, 패키지, 복합체 구조

</details>

### 056. UML 행위(동적) 다이어그램 6개를 쓰시오.

<details><summary>정답</summary>

유스케이스, 시퀀스, 커뮤니케이션, 활동, 상태, 타이밍

</details>

### 057. 유스케이스에서 include와 extend의 차이를 쓰시오.

<details><summary>정답</summary>

- **include**: 기본 유스케이스 수행 시 **반드시** 실행
- **extend**: 조건에 따라 **선택적**으로 실행

</details>

### 058. UML 관계에서 집합(◇)과 합성(◆)의 차이를 쓰시오.

<details><summary>정답</summary>

- **집합(Aggregation ◇)**: 전체가 사라져도 부분이 **독립 존재** 가능
- **합성(Composition ◆)**: 전체가 사라지면 부분도 **함께 소멸**

</details>

### 059. 클래스 다이어그램의 접근제어자를 쓰시오.

<details><summary>정답</summary>

- `+` public (공개)
- `-` private (비공개)
- `#` protected (상속만)
- `~` package (같은 패키지)

</details>

### 060. 시퀀스 다이어그램에서 라이프라인과 활성상자를 설명하시오.

<details><summary>정답</summary>

- **라이프라인**: 객체가 메모리에 존재하는 기간, 점선으로 표현
- **활성상자**: 객체가 메시지를 주고받으며 구동 중임을 직사각형으로 표현

</details>

## D. 테스트 — 15문제

### 061. 테스트 단계를 순서대로 쓰시오.

<details><summary>정답</summary>

**단위 → 통합 → 시스템 → 인수** (단통시인)

</details>

### 062. 하향식 통합에서 사용하는 것은 ( ① ), 상향식 통합에서 사용하는 것은 ( ② )이다.

<details><summary>정답</summary>

① **스텁(Stub)** — 하위 모듈 대체
② **드라이버(Driver)** — 상위 모듈 대체

</details>

### 063. 테스트 오라클 4가지를 쓰시오.

<details><summary>정답</summary>

**참(True), 샘플링(Sampling), 추정/휴리스틱(Heuristic), 일관성 검사(Consistent)**
(참샘휴일)

</details>

### 064. 커버리지를 약한 것부터 강한 순으로 쓰시오.

<details><summary>정답</summary>

**구문 → 결정(분기) → 조건 → 조건/결정 → 변경조건/결정(MC/DC) → 다중조건**
(구결조 조결변 다)

</details>

### 065. "동일한 테스트를 반복하면 더 이상 결함이 발견되지 않는 현상"을 무엇이라 하는가?

<details><summary>정답</summary>

**살충제 패러독스** (Pesticide Paradox) — 테스트 케이스를 지속적으로 보완·개선해야 함

</details>

### 066. "20%의 코드에서 80%의 결함이 발견된다"는 원리는?

<details><summary>정답</summary>

**파레토 법칙** (Pareto Principle) — 결함 집중 원리

</details>

### 067. "결함을 모두 제거해도 요구사항을 만족하지 못하면 품질이 높다고 할 수 없다"는?

<details><summary>정답</summary>

**오류-부재의 궤변** (Absence of Errors Fallacy)

</details>

### 068. 알파 테스트와 베타 테스트의 차이를 쓰시오.

<details><summary>정답</summary>

- **알파 테스트**: **개발자 장소**에서 사용자가 개발자 앞에서 수행 (통제된 환경)
- **베타 테스트**: **사용자 환경**에서 선정된 사용자가 수행 (비통제 환경)

</details>

### 069. 화이트박스 테스트와 블랙박스 테스트의 차이를 쓰시오.

<details><summary>정답</summary>

- **화이트박스**: 내부 코드 **구조** 기반, 논리적 경로 테스트 (구조 테스트)
- **블랙박스**: 외부 **명세** 기반, 기능 동작 테스트 (기능 테스트)

</details>

### 070. 블랙박스 테스트 기법 중 "입력 데이터를 유효/무효 그룹으로 나누어 대표값을 테스트"하는 기법은?

<details><summary>정답</summary>

**동치 분할 검사** (Equivalence Partitioning)

</details>

### 071. "실행 없이 소스코드를 분석하여 코딩 표준, 복잡도, 결함을 발견하는 도구"는?

<details><summary>정답</summary>

**정적 분석 도구** (Static Analysis Tools)

</details>

### 072. 회귀 테스트(Regression Test)란?

<details><summary>정답</summary>

소프트웨어의 변경 또는 수정된 코드에 **새로운 결함이 없음**을 확인하는 테스트

</details>

### 073. 검증(Verification)과 확인(Validation)의 차이를 쓰시오.

<details><summary>정답</summary>

- **검증**: **개발자** 시각, 명세서대로 만들었는지 (Are we building the product **right**?)
- **확인**: **사용자** 시각, 요구대로 만들었는지 (Are we building the **right** product?)

</details>

### 074. ★MC/DC(Modified Condition/Decision Coverage)란?

<details><summary>정답</summary>

각 개별 조건이 전체 결정(Decision)의 결과에 **독립적으로 영향**을 미치는 것을 확인하는 커버리지. 변경 조건/결정 커버리지.

</details>

### 075. 애플리케이션 성능 측정 4대 지표를 쓰시오.

<details><summary>정답</summary>

- **처리량** (Throughput): 일정 시간 내 처리하는 일의 양
- **응답 시간** (Response Time): 요청~응답 도착까지 시간
- **경과 시간** (Turn Around Time): 의뢰~처리 완료까지 시간
- **자원 사용률** (Resource Usage): CPU, 메모리, 네트워크 사용량

</details>

## E. 보안 & 네트워크 — 20문제

### 076. 보안의 3요소(CIA)를 쓰시오.

<details><summary>정답</summary>

- **기밀성** (Confidentiality): 인가된 사용자만 접근
- **무결성** (Integrity): 인가된 사용자만 수정
- **가용성** (Availability): 인가된 사용자 언제든 사용 가능

</details>

### 077. 대칭키와 비대칭키 암호화의 차이를 쓰시오.

<details><summary>정답</summary>

- **대칭키**: 암호화/복호화에 **같은 키** 사용 (AES, DES, SEED, ARIA)
- **비대칭키**: **공개키**(암호화) + **비밀키**(복호화) 쌍 사용 (RSA, ECC)

</details>

### 078. IDS와 IPS의 차이를 쓰시오.

<details><summary>정답</summary>

- **IDS** (Intrusion Detection System): 침입 **탐지**만
- **IPS** (Intrusion Prevention System): 침입 **탐지 + 차단**

</details>

### 079. 접근제어 3가지(DAC, MAC, RBAC)를 설명하시오.

<details><summary>정답</summary>

- **DAC** (Discretionary): **임의적** 접근제어, 소유자가 권한 설정
- **MAC** (Mandatory): **강제적** 접근제어, 등급 기반 (군사용)
- **RBAC** (Role-Based): **역할** 기반 접근제어

</details>

### 080. SQL Injection 공격을 설명하시오.

<details><summary>정답</summary>

웹 애플리케이션의 보안 취약점을 이용하여 **악의적인 SQL문을 삽입·실행**시켜 데이터베이스에 접근하여 정보를 탈취하거나 조작하는 공격 기법

</details>

### 081. XSS(Cross Site Scripting)를 설명하시오.

<details><summary>정답</summary>

웹 페이지에 **악의적인 스크립트를 삽입**하여 해당 페이지를 열람하는 사용자의 브라우저에서 실행되게 하여 정보를 탈취하는 공격

</details>

### 082. DoS 공격 유형 중 SYN Flooding을 설명하시오.

<details><summary>정답</summary>

공격자가 가상의 클라이언트로 위장하여 **3-way-handshake 과정을 의도적으로 중단**시킴으로써 서버가 대기 상태에 놓여 정상 서비스를 못하게 하는 공격

</details>

### 083. ★IPSec이란?

<details><summary>정답</summary>

인증헤더(AH)와 암호화(ESP)를 이용한 프로토콜로, **네트워크 계층(IP)**에서 보안성을 제공하는 표준화된 기술

</details>

### 084. OSI 7계층을 위에서 아래로 쓰시오.

<details><summary>정답</summary>

**응용 → 표현 → 세션 → 전송 → 네트워크 → 데이터링크 → 물리**
(응표세전네데물, 아래에서 위로는 물데네전세표응)

</details>

### 085. OSI 계층별 장비를 매칭하시오.

<details><summary>정답</summary>

- 1계층(물리): **허브**, 리피터
- 2계층(데이터링크): **스위치**, 브리지
- 3계층(네트워크): **라우터**
- 전 계층: **게이트웨이**

</details>

### 086. TCP와 UDP의 차이를 쓰시오.

<details><summary>정답</summary>

- **TCP**: 연결형, 신뢰성, 순서 보장, 느림 (3-way-handshake)
- **UDP**: 비연결형, 비신뢰성, 빠름, 실시간 전송에 적합

</details>

### 087. ARP와 RARP의 차이를 쓰시오.

<details><summary>정답</summary>

- **ARP**: IP 주소 → **MAC 주소** 변환
- **RARP**: MAC 주소 → **IP 주소** 변환

</details>

### 088. ★chmod 명령에서 숫자 권한을 설명하시오. chmod 751 a.txt

<details><summary>정답</summary>

- r(읽기)=**4**, w(쓰기)=**2**, x(실행)=**1**
- 7(rwx): 소유자 = 읽기+쓰기+실행
- 5(r-x): 그룹 = 읽기+실행
- 1(--x): 기타 = 실행만

</details>

### 089. IPv6의 특징과 주소 체계를 쓰시오.

<details><summary>정답</summary>

- **128비트**, 16비트씩 8부분, 콜론(:) 구분, 16진수 표현
- **유니캐스트**: 1:1 통신
- **멀티캐스트**: 1:N 통신
- **애니캐스트**: 1:가장 가까운 1 통신

</details>

### 090. 라우팅 프로토콜 RIP과 OSPF의 차이를 쓰시오.

<details><summary>정답</summary>

- **RIP**: 소규모, 최대 **홉수 15 제한**, 거리 벡터 방식
- **OSPF**: 대규모, **홉수 제한 없음**, 변화분만 전파, 링크 상태 방식

</details>

### 091. 프로토콜의 3가지 기본 요소를 쓰시오.

<details><summary>정답</summary>

- **구문** (Syntax): 데이터 형식, 부호화, 신호 레벨
- **의미** (Semantics): 제어 정보, 오류 관리
- **시간** (Timing): 통신 속도, 메시지 순서

</details>

### 092. ★랜섬웨어(Ransomware)란?

<details><summary>정답</summary>

컴퓨터에 잠입하여 내부 **파일을 암호화**하여 사용자가 열지 못하게 하고, **암호 해독 조건으로 금전을 요구**하는 악성 프로그램

</details>

### 093. 백도어(Back Door) 탐지 방법 3가지를 쓰시오.

<details><summary>정답</summary>

**무결성 검사, 로그 분석, SetUID 파일 검사**

</details>

### 094. ★제로데이 공격(Zero Day Attack)이란?

<details><summary>정답</summary>

보안 취약점이 발견되었을 때, 취약점의 존재가 공표되거나 **패치가 나오기 전에** 해당 취약점을 통해 이루어지는 공격

</details>

### 095. ★AJAX란?

<details><summary>정답</summary>

**Asynchronous JavaScript and XML**. 브라우저의 XMLHttpRequest 객체를 이용하여 전체 페이지를 새로고침하지 않고 **일부분만 비동기적으로** 데이터를 로드하는 기법

</details>

## F. OS & 기타 용어 — 5문제

### 096. 교착상태(Dead Lock) 발생의 4가지 필요충분 조건을 쓰시오.

<details><summary>정답</summary>

- **상호 배제** (Mutual Exclusion)
- **점유와 대기** (Hold and Wait)
- **비선점** (Non-preemption)
- **환형 대기** (Circular Wait)
- 암기: **상점비환**

</details>

### 097. ★RTO와 RPO를 설명하시오.

<details><summary>정답</summary>

- **RTO** (Recovery Time Objective): 업무 중단~**복구까지의 시간** 목표
- **RPO** (Recovery Point Objective): 데이터 **손실을 허용하는 시점** (얼마나 이전 데이터까지 복구할 것인가)

</details>

### 098. ★UI 설계 원칙 4가지를 쓰시오.

<details><summary>정답</summary>

- **직관성**: 쉽게 이해하고 사용
- **유효성**: 목적을 정확하고 완벽하게 달성
- **학습성**: 쉽게 배우고 익힘
- **유연성**: 요구사항 수용, 실수 최소화

</details>

### 099. ★형상관리 도구 CVS, SVN, Git 중 "분산 저장소 방식"은?

<details><summary>정답</summary>

**Git** — 원격 저장소 + 로컬 저장소에 분산 저장
- SVN = 클라이언트/서버 방식 (중앙 집중형)

</details>

### 100. ★DRM(디지털 저작권 관리)이란?

<details><summary>정답</summary>

디지털 콘텐츠의 생성, 유통, 이용 **전 과정에 걸쳐** 저작권자의 의도한 용도로만 사용되도록 관리·보호하는 기술. 기술 요소: 암호화, 키관리, 패키저, 식별, 저작권표현, 정책관리, 크랙방지, 인증

</details>
